## 🔒 Arquitetura e Dados

### Estrutura de Dados
```typescript
interface NotificationConfig {
  clinic_id: string
  notification_type: 'due_date_reminder' | 'overdue_payment' | 'approval_needed' | 'payment_completed'
  days_before_due?: number
  days_after_due?: number
  enabled: boolean
  email_enabled: boolean
  sms_enabled: boolean
  dashboard_enabled: boolean
  recipients: string[]
}

interface DashboardAlert {
  clinic_id: string
  accounts_payable_id?: string
  alert_type: 'due_soon' | 'overdue' | 'approval_needed' | 'high_priority' | 'due_today'
  priority: 'low' | 'medium' | 'high' | 'urgent'
  title: string
  message: string
  is_read: boolean
  action_url?: string
}
```

## 🚀 Funcionalidades Avançadas

### Sistema de Priorização
- **Algoritmo Inteligente**: Calcula prioridade baseado em:
  - Dias de atraso (mais atraso = mais urgente)
  - Valor do pagamento (valores altos = prioridade maior)
  - Tipo de fornecedor (críticos = prioridade alta)
  - Histórico de pagamentos

### Escalação Automática
- **Níveis Configuráveis**: 3 níveis de escalação baseados em atraso
- **Notificações Graduais**: Intensidade crescente conforme atraso
- **Override Manual**: Possibilidade de escalação manual

### Performance
- **Filtros Client-Side**: Filtros instantâneos sem chamadas ao servidor
- **Lazy Loading**: Carregamento sob demanda
- **Memoização**: Cálculos otimizados com useMemo
- **Debouncing**: Em campos de busca para melhor performance
