# Story 2.1: AI Duration Prediction Engine

## Status
Draft

## Story
**As a** clinic scheduler,
**I want** an AI system that accurately predicts treatment durations based on patient history and professional patterns,
**so that** I can optimize appointment scheduling and reduce booking conflicts and delays.

## Acceptance Criteria

1. AI model analyzes historical appointment data to predict treatment durations with ≥85% accuracy
2. System considers patient factors (age, treatment history, complexity) in duration prediction
3. Professional-specific patterns are learned and applied to duration estimates
4. Real-time duration adjustments based on current appointment progress
5. A/B testing framework compares AI predictions vs manual estimates
6. Shadow testing validates AI recommendations before full deployment
7. Duration predictions integrate seamlessly with existing appointment booking flow
8. Model performance monitoring with automatic retraining triggers
9. Fallback to manual duration estimates if AI confidence <70%
10. Treatment complexity factors automatically adjust base duration estimates

## Dev Notes

### Technical Architecture Context
[Source: architecture/tech-stack.md] - **AI Infrastructure**: Use Supabase Edge Functions for ML model serving. Integrate with existing appointment system via API.

[Source: epic-2-intelligent-scheduling.md] - **AI-Enhanced Architecture**: Shadow testing framework required for gradual AI deployment with performance protection protocols.

### Machine Learning Requirements
**Model Architecture**: 
- Feature Engineering: patient_age, treatment_history_avg, professional_efficiency_rating, treatment_complexity_score
- Target Variable: actual_duration_minutes 
- Algorithm: Gradient Boosting (XGBoost) for tabular data with time series features
- Training Pipeline: Automated retraining on new data weekly

**Performance Standards**:
- Prediction Accuracy: ≥85% within ±15 minutes
- Response Time: <200ms for prediction API
- Model Confidence: ≥70% for automatic application

### Database Schema Requirements
**AI Prediction Data Models**:
- ml_duration_predictions (appointment_id, predicted_duration, confidence_score, model_version)
- ml_model_performance (model_version, accuracy_percentage, created_at, is_active)
- ab_test_assignments (user_id, test_group, test_start_date, test_end_date)
- prediction_feedback (prediction_id, actual_duration, accuracy_score, feedback_notes)

### Integration Points
[Source: story 1.1] - **Existing Appointment System**: Build on enhanced CRUD operations. Integrate with calendar views and conflict detection.

**API Integration Points**:
- POST /api/ai/predict-duration (treatment_type, patient_id, professional_id)
- GET /api/ai/model-performance (accuracy metrics, confidence intervals)
- PUT /api/appointments/{id}/duration (real-time duration updates)
