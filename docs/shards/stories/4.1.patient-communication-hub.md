# Story 4.1: Patient Communication Hub

## Status
Draft

## Story
**As a** clinic administrator and healthcare professional,  
**I want** a comprehensive communication hub for patient interaction,  
**so that** I can efficiently manage patient communications, send notifications, and maintain professional communication standards while ensuring LGPD compliance.

## Acceptance Criteria

1. **Internal Messaging System**
   - Sistema de mensagens internas entre profissionais da clínica
   - Interface chat-like para comunicação rápida
   - Histórico de conversas organizadas por paciente/caso
   - Notificações em tempo real para novas mensagens

2. **Push & Email Notifications**
   - Sistema de notificações push para dispositivos móveis
   - Integração com email para notificações importantes
   - Templates personalizáveis para diferentes tipos de comunicação
   - Configurações de preferência do usuário para notificações

3. **Appointment Reminders**
   - Lembretes automáticos para consultas (24h, 2h antes)
   - Confirmação de presença via SMS/email/push
   - Reagendamento simplificado através do lembrete
   - Histórico de lembretes enviados e respostas

4. **Patient Portal (Web App)**
   - Portal web dedicado para pacientes
   - Acesso a histórico de consultas e tratamentos
   - Upload de documentos e fotos (antes/depois)
   - Comunicação direta com a clínica

5. **Real-time Chat**
   - Chat em tempo real entre clínica e pacientes
   - Suporte a mensagens de texto, imagens e documentos
   - Indicadores de status (online, digitando, visualizado)
   - Armazenamento seguro das conversas

6. **Communication Templates**
   - Biblioteca de templates para diferentes situações
   - Templates para pré/pós-consulta, resultados, marketing
   - Personalização automática com dados do paciente
   - Aprovação de templates pelo responsável técnico

7. **LGPD Compliance & Security**
   - Criptografia end-to-end para todas as comunicações
   - Registro de auditoria para todas as interações
   - Consentimento explícito para comunicações de marketing
   - Direito ao esquecimento e portabilidade de dados

## Dev Notes

### Relevant Architecture Information

**Technology Stack**:
- Next.js 15 with App Router for communication interfaces
- Supabase Realtime for live chat functionality
- Vercel Edge Functions for notification processing
- TanStack Query for efficient data fetching
- shadcn/ui components for consistent UI

**Security & Compliance**:
- RLS (Row Level Security) policies for patient data isolation
- End-to-end encryption using crypto-js or similar
- LGPD compliance through audit logs and consent management
- ANVISA medical software requirements compliance
- JWT-based authentication with proper session management

**Real-time Features**:
- Supabase Realtime subscriptions for chat and notifications
- WebSocket connections for live status indicators
- Optimistic updates for better user experience
- Offline support with data synchronization

**Notification Architecture**:
- Vercel Edge Functions for scalable notification processing
- Queue-based system for reliable message delivery
- Multi-channel support (email, SMS, push, in-app)
- Template-based messaging with dynamic content

### Database Schema Extensions

```sql
-- Communication tables
CREATE TABLE communications (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  sender_id UUID REFERENCES profiles(id),
  recipient_id UUID REFERENCES profiles(id),
  patient_id UUID REFERENCES patients(id),
  message_type TEXT CHECK (message_type IN ('internal', 'patient', 'appointment', 'marketing')),
  content TEXT NOT NULL,
  encrypted_content TEXT, -- For sensitive communications
  template_id UUID REFERENCES communication_templates(id),
  parent_message_id UUID REFERENCES communications(id),
  read_at TIMESTAMP WITH TIME ZONE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

CREATE TABLE communication_templates (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  category TEXT NOT NULL,
  subject TEXT,
  content TEXT NOT NULL,
  variables JSONB, -- Dynamic content placeholders
  approval_status TEXT DEFAULT 'draft',
  approved_by UUID REFERENCES profiles(id),
  created_by UUID REFERENCES profiles(id),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

CREATE TABLE notification_preferences (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES profiles(id),
  email_enabled BOOLEAN DEFAULT true,
  push_enabled BOOLEAN DEFAULT true,
  sms_enabled BOOLEAN DEFAULT false,
  appointment_reminders BOOLEAN DEFAULT true,
  marketing_communications BOOLEAN DEFAULT false,
  consent_date TIMESTAMP WITH TIME ZONE,
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

CREATE TABLE chat_sessions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  patient_id UUID REFERENCES patients(id),
  staff_id UUID REFERENCES profiles(id),
  status TEXT DEFAULT 'active',
  last_message_at TIMESTAMP WITH TIME ZONE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Audit table for LGPD compliance
CREATE TABLE communication_audit (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES profiles(id),
  action TEXT NOT NULL, -- 'create', 'read', 'update', 'delete', 'export'
  resource_type TEXT NOT NULL, -- 'message', 'template', 'notification'
  resource_id UUID,
  details JSONB,
  ip_address INET,
  user_agent TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```
